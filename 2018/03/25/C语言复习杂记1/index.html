数据类型

共6种：整形4种 + 实型2种

类型 表示范围、精度、大小
char -128~127，1字节
short -32768~32767，2字节
int -2147483648~2147483647，4字节
long 32位编译器 4字节；64位编译器 8字节
float 3.4e-38~3.4e38 精确到小数点后6位 4字节
double 1.7e-308~1.7e308 精确到小数点后15位 8字节

printf格式化转换说明符

符号 作用
%a(%A) 浮点数、十六进制数字和p-(P-)记数法(C99)
%c 字符
%d 有符号十进制整数
%0nd n：位数，0：不足位补0
%f 浮点数(包括float和doulbe)
%m.nf m：总宽度会自动扩展、补齐空格 n：小数点位数，大于小数点个数右边补齐0，小于则截断
%e(%E) 浮点数指数输出[e-(E-)记数法]
%g(%G) 浮点数不显无意义的零”0”
%i 有符号十进制整数(与%d相同)
%u 无符号十进制整数
%o 八进数 e.g. 0123
%x(%X) 十六进制整数0f(0F) e.g. 0x1234
%p 指针
%le 双精度，使用科学计数法
%s 字符串，使用时传入字符串首地址

例题

例题1：

int i = -2147483648; 求

  • -i
  • ~i
  • -1-i
  • i+1

    答案：

    • -i= -2147483648
    • ~i= 2147483647
    • -1-i= 2147483647
    • i+1= -2147483647

例题2：

问最终d的值。
过程分析：

Step a b c d
1 30 X X X
2 31 31 X X
3 32 31 X X
4 33 31 31 320

由于d是char型，大小为1字节，故320会被截断。
320的二进制为1 0100 0000，高于第8位的会被截断。
320=28+26=256+64

答案：d的值为64。

例题3：（陷阱题）

问sizeof(e)=?

分析：此题看似求e的值，实为求char类型的大小。

type size
double 8 Bytes
int 4 Bytes
float 4 Bytes
char 1 Byte
short 2 Bytes

答案：1

例题4：

问ncount=?
分析：
当a和b相加时，要将int转化为unsigned int。由于a是<0，其最高位符号位是1，在转化为unsigned int 时其符号位会参与运算，结果是一个很大的数（4294967292+1==4294967293），满足>0的条件，ncount++。
当循环几次之后，此时ncount==3，a+b为4294967295+1==0，ncount不变。当a=5时，满足a+b==6>5的条件，ncount++。

答案：4

例题5：

请问输出是什么?

分析：sizeof()运算符返回的类型是size_t，它是与机器相关的unsigned类型。在某些机器上，当int和size_t相加时，要将int转化为unsigned int。由于b是负数，其符号位是1，在转化为unsigned int 时其符号位会参与运算，结果不是0，而是一个很大的数。

答案：Yes
